## Orientações para Prototipar um Wireframe

Antes de iniciar a prototipação de um wireframe, é importante seguir algumas etapas que ajudam a organizar melhor a estrutura da interface e garantir uma boa experiência para o usuário.

### 1. Entenda os Requisitos
Antes de desenhar as telas, é necessário identificar quais elementos a interface precisa ter, como por exemplo campos de e-mail, botões de ação, menus de navegação e áreas de informação. Essa etapa ajuda a definir quais funcionalidades estarão presentes na tela.

### 2. Defina o Formato da Interface
É importante decidir para qual tipo de dispositivo a interface será desenvolvida, como **web, mobile**. Essa escolha influencia a organização dos elementos e o layout das telas.

### 3. Utilize Ferramentas de Prototipação
Para criar wireframes é possível utilizar ferramentas como **Whimsical**, **Miro**, **Figma**, ou até mesmo **papel e caneta** para fazer rascunhos iniciais. O objetivo é representar visualmente a estrutura da interface de forma rápida.

### 4. Foque na Estrutura
Durante a criação do wireframe, o foco deve estar na **organização dos elementos da tela**, na **hierarquia das informações** e na **navegação entre as páginas**. Não é necessário se preocupar com cores, imagens ou detalhes visuais, pois o wireframe representa apenas a estrutura da interface.

## O Impacto do Design Centrado no Usuário

O **Design Centrado no Usuário** é uma abordagem fundamental no desenvolvimento de sistemas, pois busca colocar as necessidades e o comportamento das pessoas no centro do processo de criação. Quando a interface é projetada pensando no usuário, evitamos sistemas confusos ou difíceis de utilizar. Isso contribui para criar soluções mais **intuitivas, acessíveis e inclusivas**, permitindo que diferentes pessoas consigam utilizar a tecnologia com facilidade.

No contexto do sistema desenvolvido neste projeto, essa abordagem permite que responsáveis consigam acompanhar informações importantes, como o desempenho escolar de seus filhos, de forma **simples, clara e eficiente**, demonstrando como boas práticas de Interface Humano-Computador podem impactar positivamente o uso da tecnologia no cotidiano.
